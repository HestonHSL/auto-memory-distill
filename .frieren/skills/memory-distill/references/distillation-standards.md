# Distillation Standards (提纯整理标准)

> **定位**：Layer 2 (Memory) 提纯指南。用于在结算或话题转移时，将原始碎片升华为可执行的项目规则。

---

## 1. 提纯步骤 (Distillation Process)

1.  **价值判定**：应用本文第 2 节的质量门控进行筛选。
2.  **抽象规则**：将具体的变动（如“把 index.js 第 5 行改为...”）升华为抽象指令（如“项目中使用 A 库替代 B 库”）。
3.  **去重合并**：与 `.memory/` 现有规则比对，解决冲突。
4.  **分类入库**：更新对应的分类文件（`project-spec.md` 或 `module-[name].md`）。

---

## 2. 质量门控 (Quality Gate Checklist)

写入 Layer 2 前必须满足：

- [ ] **去具体化**：移除文件路径、行号、具体变量名。
- [ ] **高可执行性**：规则必须是 AI 读完后能立刻执行的“Prompt 指令”。
- [ ] **跨任务通用**：该规则在其他类似场景中依然有效。
- [ ] **抗衰减**：规则逻辑清晰，不会随时间推移而失效。

---

## 3. 冲突协议 (Conflict Protocol)

当新旧记忆冲突时：

- **规则演进**：若用户明确推翻旧习惯，则覆盖旧规则。
- **场景分支**：若两个规则在不同场景下均有效，则合并并详细描述各自的“触发场景”。

---

## 4. Layer 2 写入模板

```markdown
### [技术关键词标题]

- **场景描述**: [何时适用]
- **规则 (可执行)**: [指令式规则]
- **原因 / 强制性**: [逻辑背景与豁免条件]
```
