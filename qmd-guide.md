# QMD (Query Markdown) 技术指南

QMD 是一款专为 AI 记忆系统设计的语义化检索工具。它通过将 Markdown 文档向量化，实现“按需唤醒”的智能记忆检索。

## 1. 核心原理

QMD 的运作基于 **RAG (Retrieval-Augmented Generation)** 架构，分为三个核心阶段：

### 向量化 (Embedding)

将文本转化为高维向量（坐标）。

- **语义接近 = 距离接近**：即使措辞不同（如“外边距”与“间距”），只要语义相似，它们在向量空间中的距离就会很近。

### 智能分块 (Chunking)

针对 Markdown 结构进行优化。

- QMD 会识别文件中的 `H1`、`H2`、`H3` 标签，将每个小节作为一个独立的语义单元。这样在检索时，AI 能够获得最精确的代码片段或规则，而不是整个大文件。

### 语义检索 (Semantic Search)

- **Top-K 召回**：根据用户当前的请求，计算其向量与数据库中所有分块向量的距离，返回最相关的数条记录。

## 2. 三层架构中的角色

在 Memory-Distill 系统中，QMD 充当了“索引搜索层”：

1. **Layer 1 (.logs/)**: 记录原始碎片。
2. **Layer 2 (.memory/)**: 存储提纯后的规则。
3. **Layer 3 (QMD Index)**: 将 Layer 2 的规则映射到语义空间，实现瞬间召回。

## 3. 常用命令

### 初始化索引

对记忆库进行全文扫描并建立向量索引。

```bash
qmd embed .memory/
```

### 语义查询

根据自然语言描述查找最相关的规则。

```bash
qmd query "如何处理表单边距？"
```

### 增量更新

当 `.memory/` 下的文件发生变动时，建议重新运行 `embed` 命令以维持索引的一致性。

## 4. 为什么使用 QMD？

- **极省 Token**：AI 只读取最相关的几行规则，而不是加载整个记忆库。
- **抗干扰强**：即使记忆库文件非常多，AI 也能精准定位，不会因为上下文过长而产生幻觉（Hallucination）。
- **零手动索引**：不需要维护 `index.md` 里的关键词，QMD 自动理解内容。
