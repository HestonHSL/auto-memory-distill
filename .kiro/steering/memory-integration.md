# 记忆系统集成提醒

## 核心目标
**捕获重复发生的纠错和修改行为，提炼成规则，减少未来的重复纠错**

## 执行流程

### 对话开始
1. 检查 `.kiro/skills/auto-memory-rules/memory/` 目录
2. 根据关键词搜索相关规则
3. 将规则应用到当前上下文

### 对话执行（核心）
**持续监听并立即捕获：**
- 用户的纠错行为（"不对"、"应该是..."）
- 用户的修改要求（"改成..."、"换成..."）
- 重复出现的问题模式
- 用户表达的偏好（"我更喜欢..."、"应该用..."）

→ 立即创建 fragment 记录

### 对话结束
1. 检查 fragments 目录
2. 识别重复模式，提纯为 rule
3. 更新索引并生成 checklist

## 核心原则
- **用户当前请求 > 历史记忆规则**
- **重复纠错 = 规则信号** - 2次以上的相同纠错必须记录
- 详细逻辑参考 `.kiro/skills/auto-memory-rules/SKILL.md`
